Building a Crypto Currency Tracker in Salesforce with LWC. 📚

Introduction

May 12th 2022 saw the price of Terra (LUNA) fall 96% in a day. People lost life savings overnight. The knock on effect sent shockwaves through the crytpo market and saw the prices of Bitcoin, Etherium and Coinbase all down big. So much so that even Binance iteself had to suspend trading. For more information on the LUNA crash checkout the follow article: The Rise and Fall of LUNA. The crash got my thinking about price and market visibilty for crypto assets whilst working on internal networks and applications. Could we expose an integration in Salesforce that would allow someone to quickly scan or check for an asset using realtime data? For a while I had been wanting to learn some LWC (Lighting Web Components), so I though I would try and build a small application that meets this requirement.

Usage

The Cyrpto Tracker Application allows a user to search for a specific ticker. For example BTC (Bitcoin).

Lastly, the user will be able to see the top trending coins for the last 24 hours scrolling along the bottom of the application. The data is a snapshot of each specific trending coin. Which includes the ticker, the price (USD) and the hourly percentage change.

Technical Details

The Crypto Tracker Application is hosted on a public page on a Salesforce community in my developer edition org. When it first loads, it authenticates with the two API's I am using to fetch the realtime data: CryptoCompare and BraveNewCoin. I am storing the client id's and secrets in a custom setting within Salesforce.

After authenticating a call is made to CryptoCompare to fetch the trending coins for the past 24 hours. The data is deserialized into a APEX response class, formatted in the js and passed through to the front end in the form of an array. This can then be passed in a foreach within a marquee tag. (Redundant I know, but for the sake of a small learning project I did not have the patience for custom css) 😭.

On user input there is handler function which listens for keyup events. Once a user hits enter or the input length satisfies (>=3) a call is made to BraveNewCoin. Which returns some data including a coin id (content.id). ****Please see Postman screenshot below.

If successful, the response is chained in a Promise(). Which then uses this content.id in a subsequent request to BraveNewCoin to fetch price/market data. *** Please see Postman screenshot below. Data is deserialized as before and returned with getters to the page.
